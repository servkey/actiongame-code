Setting up a Server

Command Line Options

In the file actioncube_server.bat (or the respective .sh file in linux) you can add or change command line options. The possible command line options can be optained from the cube config documentation. ActionCube behaves mostly like Cube, there are a few differences though:

  • Unline cube, ActionCube uses the ports 28764 and 28766 (UDP)

  • There is the additional command line option -r to specify a maprotation file:

    -rconfig/maprot.cfg

    See chapter maprotation below for more information.

Maprotation

In the maprotation file specified on the command line (default: config/maprot.cfg), you can specify the map, mode, time and voting for your server. Each line defines one round. Example: 

// server side maprotations
// WARNING: You can NOT use CubeScript in there
// use the following scheme:
// map:mode:time:allowVote

// tdm
ac_complex:0:10:1
ac_depot:0:10:1
ac_desert:0:10:1
ac_mines:0:10:1

// ctf
ac_mines:5:10:1
ac_depot:5:10:1

The first line, ac_complex:0:10:1, means, the map ac_complex is played in mode 0 (Team Deathmatch) for 10 minutes. The connected players are allowed to vote.
