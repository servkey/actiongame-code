Setting up a Server

Command Line Options

In the file actioncube_server.bat (or the respective .sh file in linux) you can add or change command line options. The possible command line options can be obtained from the cube config documentation. ActionCube behaves mostly like Cube, there are a few differences though:

  • Unlike cube, ActionCube uses the ports 28763 and 28764 (UDP)
  • The command line option -x specifies the master password:
    -xFOOBAR
  • The command line option -r specifies a maprotation file:
    -rconfig/maprot.cfg

See the next chapter for more information about the effect of new command line options.

Maprotation

In the maprotation file specified on the command line (default: config/maprot.cfg), you can specify the map, mode, time and voting for your server. Each line defines one round. Example: 

// server side maprotations
// WARNING: You can NOT use CubeScript in there
// use the following scheme:
// map:mode:time:allowVote

// tdm
ac_complex:0:10:1
ac_depot:0:10:1
ac_desert:0:10:1
ac_mines:0:10:1

// ctf
ac_mines:5:10:1
ac_depot:5:10:1

The first line, ac_complex:0:10:1, means, the map ac_complex is played in mode 0 (Team Deathmatch) for 10 minutes. The connected players are allowed to vote.

Administration

Master

ActionCube comes with a very simple implementation of a master console. If you set a master password for your server, you can connect to it and login using this command:

masterlogin FOOBAR

where FOOBAR is the password you set. If your login attempt fails, you will be kicked from the server.

Kick and Ban

Once you are logged in as master, you can kick and ban people from the server. Type this command into the console to get the master menu that lets you select the players to kick/ban:

master

Banning players means, any connection attempts from their IP will be refused for the next 20 minutes. To remove the bans, select Remove Bans in the master menu.
